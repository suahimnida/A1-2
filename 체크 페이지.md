인터넷 정보를 받아와서 여행지 추천해주는 파이썬 프로그램 만들기

- 여러 API를 엮은 인사이트 제작
- Python에서 API 호출 -> LLM API가 여행지 추천 -> 지도 API가 위치 추적 -> 최종 리포트 생성
- LLM API와 지도 API를 조합하여 국내 여행 추천 프로그램 개발
- 사용자가 여행 날짜 입력 -> LLM 해당 시기에 여행하기 좋은 지역 추천 -> 지도 API로 맛집 정보 검색 -> 최종 여행 리포트 생성
- 단순 하나의 API를 호출하는 것을 넘어, 여러 API의 데이터를 조합하고 AI로 인사이트를 도출하는 프로그램 제작

**과제 목표**
REST API의 요청/응답 구조와 HTTP 메서드(GET/POST)의 차이를 말로 설명
LLM 출력 결과를 구조화(JSON)하여 다음 단계(지도/장소 검색)의 입력으로 활용하는 흐름 설명
외부 API 호출에서 발생하는 대표 오류(인증/쿼터/네트워크/파싱)와 대응 원칙 설명
API 키를 코드에서 직접 작성하지 않고 .env/환경변수로 관리하는 이유 설명

**기능 요구 사항**

0) 기본
Python 3.10 이상 사용
터미널에서 실행 가능해야함 (웹 UI 구현은 요구X)

1) 환경변수 설정 예시
macOS/Linux (현재 터미널 세션에만 적용): export OPENAI_API_KEY="YOUR_KEY"
Window Powershell (현재 세션에만 적용): $env:OPENAO_API_KEY="YOUR_KEY"

3) CLI 인터페이스
argparse를 활용하여 CLI로 실행
필수 옵션: -date "YYYY-MM-DD"
입력값 검증: 날짜 형식이 올바르지 않으면 사용법을 출력하고 종료

4) API 제공자 선택 규칙
LLM API는 택1로 구현(OpenAI 계열 API / Google Gemini 계열 API)
지도/장소 검색 API는 택 1로 구현(Kakao Local(키워드/카테고리 기반 장소 검색) / Naver Local Search(지역/장소 검색)) -> 국내 장소 검색 가능해야 함
*다른 제공자 활용 시, 장소 검색이 가능하고, 응답을 JSON으로 받을 수 있어야 하며, 최소 응답 필드 혹은 이에 준하는 값을 확보할 수 있어야함
 : place_name, address, lat, ing(또는 x/y), url(가능한 경우)

5) LLM API 연동 - 날씨/행사 정보
입력: 사용자가 입력한 date
출력: 반드시 JSON으로 파싱 가능한 텍스트를 생성하도록 프롬프트 설계
1차 JSON 최소 스키마 (필수 키/타입)
: recommended_city: string(예: "제주", "강릉"), weater: string(해당 시기 일반적 날씨 요약),
  events: array of string (행사/축제 후보 1~3개), reason: string(추천 근거 2~4문장)
실제 날씨/행사 데이터의 "정확도"를 평가하는 미션이 아니며, 중요한 것은 "구조화된 출력"과 "다음 단계 입력으로의 연결"

6) 지도/장소 검색 API 연동 - 맛집 검색
입력: 1차 JSON의 recommended_city
출력: 해당 도시 또는 키워드 기준으로 맛집 N곳 (권장 5곳)을 검색
맛집 아이템 최소 필드
: name: string, address: string, category: string(가능한 경우), url: string(가능한 경우), x, y 또는 lat, ing: number(가능한 경우)
인증/요청 최소 가이드
: Kakao Local 예시 _ 인증 헤더에 키를 실어 요청하는 방식 (401/403 발생 시 키/권한/도메인 설정을 점검)
  Naver Local Search 예시 _ 클라이언트 ID/Secret을 헤더로 보내는 방식 (401/403 발생 시 키 값/헤더명 오타를 점검)
검색 결과가 0건이면 프로그램이 중단되지 않아야 하며, "데이터 없음" 상태로 다음 단계(리포트 생성)로 진행

7) LLM API 연동 - 최종 리포트 생성
입력: 1차 추천 JSON + 맛집 목록 (0건 일 수 있음)
출력: 최종 여행 리포트를 Markdown 텍스트로 생성
리포트에는 추천 지역 + 추천 이유 요약 / 날씨 요약 / 행사/축제 목록 / 맛집 리스트 (0건이며 "데이터 없음"으로 표기) / 1일 일정 제안(오전/오후/저녁 정도의 수준)

8) 에러 처리
try-except로 API 호출/파싱 오류를 처리
최소 정책
: API 키 미설정: 즉시 종료 + 설정 방법 안내 출력
  지도/장소 API 실패(네트워크/인증/쿼터 등): 맛집 섹션을 "데이터 없음" 처리하고 리포트 생성은 계속 진행
  LLM JSON 파싱 실패: 재요청 1회 또는 "필수 키만 다시 JSON으로 출력"하도록 프롬프트를 수정해 재시도 1회
실패가 발생하면, 결과 리포트에 errors 섹션으로 요약을 남길 수 있도록 내부적으로 오류 목록을 관리 (빈 리스트여도 무방)

9) API 키 관리 (보안)
API 키를 코드/README/결과물에 직접 작성하지 않음
환경변수 또는 .env 파일 (예: python-dotenv 사용 가능)에서 키를 읽어옴
: 협업/공유 시 실수로 키가 공개되는 것을 막음
  키를 교체하더라도 코드를 수정하지 않아도 됨 (운영/배포에 유리)
  과금/쿼터가ㅏ 걸린 서비스에서 사고를 예방
제출물(README/로그/결과 파일)에 키가 노출되지 않도록 주의

10) 결과 저장
results/ 폴더를 생성하고, 실행 날짜 기준으로 파일 저장
원본 데이터 JSON에는 1차 추천 JSON(파싱 결과), 맛집 검색 결과(리스트, 0건 가능), 오류 요약(errors: array) 포함
최종 리포트는 .md 파일로 저장

1. CLI 기반 Python 프로그램

입력/요청: -date "YYYY-MM-DD" (필수)
출력/화면: 진행 로그 + 결과 저장 경로 안내

2. 실행 결과 데이터

출력/화면: results/ 폴더에 아래 파일 생성
원본 데이터 JSON 1개 이상 (1차 추천 결과 + 맛집 검색 결과 포함)
최종 여행 리포트 markdown 1개

3. README.md

프로그램 개요, 실행 방법, API 키 설정 방법, 결과물 확인 방법
API 키가 유출되지 않도록 주의 사항 포함
